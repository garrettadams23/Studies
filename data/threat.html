⚔️ Cyber Kill Chain — Lockheed Martin 7-PHASE MODEL ▶
Attack Framework
7 Phases of a Targeted Cyberattack
Breaking the chain at any phase stops the attack. Earlier disruption = lower adversary ROI. Defenders map controls to each phase to identify detection and prevention gaps.
01
Reconnaissance
OSINT, DNS/WHOIS, Shodan, LinkedIn scraping, passive & active scanning of target infrastructure
02
Weaponization
Create exploit + delivery vehicle: malware dropper, infected PDF/Office macro, exploit kit
03
Delivery
Phishing email, USB drop, watering hole attack, web drive-by, compromised supply chain
04
Exploitation
Trigger exploit on target: CVE, macro execution, browser vulnerability, social engineering success
05
Installation
Install RAT, backdoor, rootkit for persistence. Modify registry, cron jobs, WMI subscriptions
06
C2 (Command & Control)
Establish covert channel to attacker's C2: HTTP/S beaconing, IRC, DNS tunneling, social media
07
Actions on Objectives
Exfiltrate data, ransomware deployment, destructive wiper, lateral movement, espionage, sabotage

🎭 Social Engineering Techniques ATTACK TAXONOMY ▶
Reference — Source: CEH + A+ Cheat Sheets
Human-Based · Computer-Based · SE Attack Types
Social engineering exploits human psychology rather than technical vulnerabilities. It is the leading initial access vector. Awareness training is the primary defense.
▸ COMPUTER-BASED TECHNIQUES
Technique Description Target
Phishing Mass deceptive email appearing legitimate; harvest credentials General employees
Spear Phishing Targeted phishing using personal information about the victim Specific individual
Whaling Phishing targeting high-value executives (CEO, CFO) C-suite executives
Vishing Voice phishing by telephone or voicemail Phone users
Smishing SMS/text-based phishing with malicious link Mobile users
Pharming DNS poisoning or hosts file modification to redirect to evil twin website Web users
Evil Twin Rogue Wi-Fi AP mimicking legitimate SSID to capture traffic Wi-Fi users
▸ HUMAN-BASED TECHNIQUES
Technique Description
Impersonation Pose as IT support, vendor, contractor or authority figure to gain access
Tailgating / Piggybacking Follow authorized person through a secure door without badge-in
Shoulder Surfing Observe victim entering credentials; can use recording device at distance
Dumpster Diving Recover sensitive documents, credentials, hardware from trash
Pretexting Create a fabricated scenario (pretext) to manipulate victim into disclosure
Quid Pro Quo Offer a service/benefit in exchange for information (e.g., fake IT help)
🦠 Malware Types Reference TAXONOMY TABLE ▶
Reference — Source: A+ + CEH Cheat Sheets
Virus · Worm · Trojan · Rootkit · Ransomware · Spyware
▸ MALWARE CATEGORIES
Type Description Key Characteristic
Virus Self-replicating code that attaches to legitimate files; needs host to spread Requires user action to trigger
Worm Self-replicating; spreads across networks independently without host file No user action needed; uses network vulns
Trojan Horse Appears useful/legitimate; hides malicious payload behind benign functionality No self-replication; user installs willingly
Rootkit Gains kernel/admin level access; hides presence by modifying OS Extremely difficult to detect and remove
Ransomware Encrypts files/system; demands payment for decryption key Financially motivated; backup is best defense
Spyware Silently monitors and records activity; keyloggers, screen capture, browser hijack Privacy violation; often bundled with freeware
Botnet / Bot Compromised systems under attacker C2 control for DDoS, spam, mining Part of larger coordinated network
Logic Bomb Dormant code triggered by a specific condition (date, event, user action) Insider threat; hard to detect before trigger
Fileless Malware Runs in memory; no files written to disk; uses LOLBins (PowerShell, WMI) Evades traditional AV; leaves minimal traces
▸ VIRUS SUBTYPES (from CEH)
Subtype Description
Boot Sector Moves/overwrites MBR; loads before OS; almost impossible to remove without reimaging
Macro Written in MS Office macro language (VBA); triggers when document is opened
Polymorphic Constantly changes its signature/code to evade signature-based AV detection
Metamorphic Completely rewrites itself when infecting each new file; harder to detect than polymorphic
Multipartite Simultaneously infects boot sector AND executable files
Stealth Actively hides itself by intercepting OS calls that would reveal its presence
Cavity Hides in empty space within executable files; does not increase file size
💣 DoS / DDoS & Network Attack Types ATTACK REFERENCE ▶
Reference — Source: CEH Cheat Sheet
Denial of Service · Buffer Overflow · MITM · Injection
▸ DOS / DDOS ATTACK TYPES
Attack Mechanism Defense
SYN Flood Send thousands of SYN packets with spoofed IP; target runs out of half-open connections SYN cookies, rate limiting
ICMP Flood Flood target with ICMP echo requests with spoofed source; exhaust bandwidth ICMP rate limiting
Smurf Attack Send ICMP broadcast to subnet with target's spoofed source IP; whole subnet responds to target Disable directed broadcast
Fraggle Attack Like Smurf but uses UDP instead of ICMP Block UDP broadcast
Ping of Death Send oversized ICMP packets (>65,535 bytes fragmented); system crashes on reassembly Modern OS patched
Application Level Overwhelm specific app (HTTP GET flood, Slowloris); target can't handle requests WAF, rate limiting, CDN
Amplification Use DNS/NTP servers to amplify small request into large response aimed at victim BCP38 ingress filtering
▸ BUFFER OVERFLOW TYPES
Type Description Dangerous Functions (C)
Stack Overflow local variables on the call stack; overwrite return address to redirect execution gets(), strcpy(), strcat(), printf()
Heap Overflow dynamically allocated memory; overwrite function pointers or object data malloc() misuse, manual memory mgmt
