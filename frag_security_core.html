🛡️ CIA Triad & Core Security Concepts SECURITY+ · SECURITYX ▶
Reference — Foundational Security Principles
CIA · AAA · Defense in Depth · Zero Trust · Least Privilege
The CIA Triad is the foundation of all information security. Every control, threat, and countermeasure maps to one or more of its three properties.
▸ CIA TRIAD
Confidentiality
Data is accessible only to authorized parties.
Controls: encryption, access control, MFA, classification
Threats: eavesdropping, data exfiltration, sniffing
Integrity
Data is accurate and unaltered unless authorized.
Controls: hashing (SHA-256), MACs, digital signatures, checksums
Threats: tampering, MITM, injection, corruption
Availability
Systems and data are accessible when needed.
Controls: redundancy, backups, failover, DDoS mitigation
Threats: DDoS, ransomware, hardware failure, disasters
▸ AAA FRAMEWORK
ComponentDefinitionExamples
AuthenticationVerify identity — prove who you arePassword, MFA, biometrics, certificates, RADIUS
AuthorizationGrant permissions — what you can doACLs, RBAC, ABAC, firewall rules, sudo
AccountingTrack actions — what you didLogs, audit trails, SIEM, syslog, session records
▸ SECURITY PRINCIPLES
PrincipleDescription
Least PrivilegeGrant only the minimum access required for the task; revoke when done
Need to KnowAccess to data only when operationally necessary, even if privileged
Separation of DutiesNo single person controls a complete critical process end-to-end
Defense in DepthLayered controls so no single failure causes total compromise
Zero Trust"Never trust, always verify" — authenticate and authorize every request regardless of network location
Fail Secure / Fail SafeFail secure = deny on failure (firewall drops traffic); Fail safe = preserve life/safety (doors unlock on fire)
Security by ObscurityAlone is NOT a valid control — cannot be the only defense
Non-repudiationPrevent denial of actions — achieved via digital signatures + audit logs

Reference — Authentication Methods & Protocols
Authentication Factors · MFA · SSO · SAML · OAuth · OIDC
Strong authentication requires multiple factors from different categories. 2FA = two factors. MFA = two or more. The factor categories cannot overlap in the same solution.

🔐 Cryptography — Algorithms, PKI & Protocols SECURITY+ · SECURITYX ▶
Reference — Modern Cryptography
Symmetric · Asymmetric · Hashing · PKI · TLS · Common Algorithms
Symmetric = same key encrypt/decrypt (fast, shared key problem). Asymmetric = public/private key pair (slow, solves key exchange). Hashing = one-way digest (integrity, not reversible). In practice: asymmetric exchanges the symmetric key; symmetric encrypts bulk data.

Malware Types · Social Engineering · Network Attacks · Web Attacks
