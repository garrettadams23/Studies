URL / Percent Encoding — Rules, Characters & Interactive Decoder
URLs can only contain a safe subset of ASCII characters. Any character outside that set must be percent-encoded: replaced with %XX where XX is the two-digit uppercase hex value of the UTF-8 byte. Also called percent-encoding (RFC 3986).
▸ ENCODING RULES
Safe Characters
Never encoded: A–Z a–z 0–9 - _ . ~
Also safe in specific URL parts: ! * ' ( )
Reserved Characters
Have special URL meaning — encode when used as data:
: / ? # [ ] @ ! $ & ' ( ) * + , ; =
Always Encode
Spaces, Unicode, control chars, and anything outside ASCII printable range must always be percent-encoded. Space → %20 (or + in query strings only)
Query String Rule
In application/x-www-form-urlencoded (HTML forms), space becomes +. In all other contexts use %20.

▸ URL STRUCTURE — WHICH PARTS GET ENCODED
# Full URL anatomy https://user:pass@example.com:8080/path/to/page?key=val&x=1#anchor │ │ │ │ │ │ │ │ scheme │ │ host port path query fragment │ │ user password ← encode : and @ when used as data here # Path segment — encode everything except unreserved + sub-delimiters https://example.com/search/hello%20world # space in path → %20 https://example.com/file/report%232024.pdf # literal # → %23 # Query string — key=value pairs, separate with & https://example.com/search?q=C%2B%2B&lang=en # C++ → C%2B%2B https://example.com/api?name=John+Doe # space in form → + https://example.com/api?name=John%20Doe # space in URL → %20 # Fragment (hash) — not sent to server https://example.com/page#section-2 # fragment rarely needs encoding
▸ ENCODING IN CODE
## JavaScript encodeURIComponent("hello world & more") # → "hello%20world%20%26%20more" (encodes everything except unreserved) encodeURI("https://example.com/path with spaces") # → "https://example.com/path%20with%20spaces" (preserves URL structure chars) decodeURIComponent("%48%65%6C%6C%6F") # → "Hello" ## Python 3 from urllib.parse import quote, quote_plus, unquote quote("hello world/path") # → 'hello%20world/path' (/ safe by default) quote("hello world/path", safe='') # → 'hello%20world%2Fpath' (encode / too) quote_plus("hello world") # → 'hello+world' (form encoding) unquote('hello%20world') # → 'hello world' ## Bash / curl curl --data-urlencode "param=hello world & more" https://example.com python3 -c "import urllib.parse; print(urllib.parse.quote('hello world'))"
