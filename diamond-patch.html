💎 Diamond Model of Intrusion Analysis ANALYSIS MODEL ▶
Framework
Adversary · Infrastructure · Capability · Victim — 4 Numbered Steps
Every intrusion event has four core features. Analysts pivot between features to map entire campaigns. Meta-features: timestamps, phase, direction, results, methodology. Developed by Caltagirone, Pendergast & Betz (2013).
▸ DIAMOND MODEL — STEP DIAGRAM WITH CONNECTING LINES
▸ STEP-BY-STEP ANALYSIS
1
Adversary
Identify who — threat actor, nation-state, hacktivist, or insider. Attribute using TTPs, infrastructure overlap, and malware fingerprints.
2
Infrastructure
Map what they used — C2 servers, IP ranges, domain registrations, bulletproof hosting. Pivot to find related campaigns.
3
Capability
Analyze how they attacked — malware families, CVE exploits, custom tools, LOLBins, and MITRE ATT&CK TTP mapping.
4
Victim
Characterize who was targeted — org type, sector, geography, specific assets. Understand motive: data, disruption, espionage.
Pivot Technique: Any known feature reveals unknown features. Known malware hash (Capability) → reveals C2 server (Infrastructure) → finds other victims (Victim) → identifies threat group (Adversary).
