📦 JSON & jq — Data Format & CLI Processor DATA FORMAT ▶
Reference — JSON RFC 8259 & jq 1.7
JSON Syntax · Data Types · jq Filters & Transforms
JSON (JavaScript Object Notation) is the universal API data format. jq is the standard CLI tool for parsing and transforming JSON. Install: sudo apt install jq · brew install jq.
▸ JSON SYNTAX & TYPES
{ "string": "hello world", "number": 42, "float": 3.14, "bool": true, "null": null, "array": [1, "two", false, null], "object": { "nested": true, "count": 3 } } // Rules: keys must be double-quoted strings, // trailing commas are NOT allowed, // comments are NOT part of JSON spec.
▸ JQ FILTERS
# Identity — pretty-print jq '.' data.json # Field access jq '.name' # top-level field jq '.user.email' # nested field jq '.items[0]' # first array element jq '.items[-1]' # last element jq '.items[1:3]' # slice index 1–2 jq '.items[]' # iterate all elements jq '.items[].name' # field from each element # Pipe and transform jq '.users[] | .email' # extract field from each jq '.[] | select(.active == true)' # filter — keep matching jq 'map(.score * 2)' # transform every element jq 'map(select(.age > 18))' # filter array jq '[.[] | select(.role=="admin")]' # collect into new array # Construct new shapes jq '{name: .name, email: .email}' # object construction jq '[.users[] | {id, name}]' # array of objects # Aggregation jq 'length' # count keys or elements jq '[.[] | .price] | add' # sum all prices jq '[.[] | .score] | max' # max value jq 'group_by(.category)' # group array by field jq 'sort_by(.name)' # sort by field jq 'unique_by(.id)' # deduplicate by field # String operations jq '.name | ascii_upcase' # uppercase jq '.items | join(", ")' # join array to string jq '"prefix_\(.name)"' # string interpolation # Useful flags jq -r '.name' # -r raw output (no quotes) jq -c '.' # -c compact (one line) jq -n '{a:1}' # -n null input (construct without file) jq --arg key val '.name == $key' # pass shell var as jq var

Reference — ANSI SQL / MySQL / PostgreSQL
DDL · DML · DQL · Joins · Aggregates · Window Functions
SQL has four main categories: DDL (schema) · DML (data) · DQL (query) · DCL (permissions). Execution order: FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY → LIMIT.
▸ DDL — SCHEMA DEFINITION
CREATE TABLE users ( id SERIAL PRIMARY KEY, email VARCHAR(255) NOT NULL UNIQUE, name TEXT NOT NULL, role TEXT DEFAULT 'user', created_at TIMESTAMPTZ DEFAULT NOW() ); ALTER TABLE users ADD COLUMN age INT; ALTER TABLE users DROP COLUMN age; CREATE INDEX idx_email ON users(email); DROP TABLE IF EXISTS temp_data; TRUNCATE TABLE logs; -- delete all rows, keep structure
▸ DML — DATA MANIPULATION
-- INSERT INSERT INTO users (email, name, role) VALUES ('alice@example.com', 'Alice', 'admin'); -- INSERT multiple rows INSERT INTO tags (name) VALUES ('sql'), ('data'), ('query'); -- UPDATE UPDATE users SET role = 'admin' WHERE email = 'alice@example.com'; -- DELETE DELETE FROM users WHERE created_at < NOW() - INTERVAL '90 days'; -- UPSERT (PostgreSQL) INSERT INTO users (email, name) VALUES ('bob@x.com', 'Bob') ON CONFLICT (email) DO UPDATE SET name = EXCLUDED.name;
▸ DQL — QUERYING
-- Basic SELECT SELECT id, name, email FROM users WHERE role = 'admin' AND created_at > '2024-01-01' ORDER BY name ASC LIMIT 25 OFFSET 0; -- Pattern matching SELECT * FROM users WHERE name LIKE 'A%'; -- starts with A SELECT * FROM users WHERE email ~* '@gmail'; -- regex (PostgreSQL) -- Aggregate functions SELECT role, COUNT(*) AS total, AVG(age) AS avg_age, MAX(age) AS oldest FROM users GROUP BY role HAVING COUNT(*) > 5 ORDER BY total DESC;
▸ JOINS
-- INNER JOIN — only matching rows SELECT u.name, o.total FROM users u INNER JOIN orders o ON o.user_id = u.id; -- LEFT JOIN — all users, nulls if no order SELECT u.name, COUNT(o.id) AS order_count FROM users u LEFT JOIN orders o ON o.user_id = u.id GROUP BY u.name; -- Subquery SELECT * FROM users WHERE id IN (SELECT user_id FROM orders WHERE total > 100); -- CTE (Common Table Expression) WITH big_orders AS ( SELECT user_id, SUM(total) AS spent FROM orders GROUP BY user_id HAVING SUM(total) > 500 ) SELECT u.name, b.spent FROM users u JOIN big_orders b ON b.user_id = u.id;
▸ WINDOW FUNCTIONS (PostgreSQL)
-- ROW_NUMBER, RANK, DENSE_RANK SELECT name, score, RANK() OVER (ORDER BY score DESC) AS rank, ROW_NUMBER() OVER (ORDER BY score DESC) AS row_num, LAG(score, 1) OVER (ORDER BY score DESC) AS prev_score, SUM(score) OVER (PARTITION BY team) AS team_total FROM leaderboard;
