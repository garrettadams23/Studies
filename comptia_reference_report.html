6-Step CompTIA Troubleshooting Process
The standard methodology tested across all CompTIA certifications. Applies to hardware, software, networking, and security issues. Each step is sequential but the process can loop back to Step 3 if a theory doesn't resolve the issue.
▸ STEP-BY-STEP FLOWCHART

▼
STEP 2
Establish a Theory of Probable Cause

▼
STEP 3
Test the Theory to Determine the Cause
Confirm theory → proceed · Theory not confirmed → reestablish theory or escalate
▼
STEP 4
Establish a Plan of Action and Implement the Solution

▼

▼
STEP 6

Create knowledge base entry · Inform stakeholders · Close ticket
💻 PC Boot Process (POST → OS) BOOT SEQUENCE ▶
Sequence Diagram
Power-On Self-Test & System Initialization
The boot process describes how a computer transitions from powered-off state to a running OS. Understanding POST errors (beep codes, error messages) is critical for A+ hardware troubleshooting.
▸ BOOT SEQUENCE
1 — Power On / Reset
CPU initializes · Sets instruction pointer to firmware entry address
2 — POST (Power-On Self-Test)
BIOS/UEFI checks CPU, RAM, GPU, storage controllers · Beep codes signal failures
3 — BIOS / UEFI Initialization
Loads hardware config · Reads CMOS/NVRAM settings · Applies boot device priority
4 — MBR / GPT Boot Sector Read
Reads partition table on boot device · Locates active partition / EFI System Partition
5 — Bootloader (GRUB / Windows Boot Manager)
Loads OS kernel from disk into RAM · May present multi-boot menu
6 — OS Kernel Load
Kernel decompresses & initializes · Sets up memory management, processes, drivers
7 — User Space / Login Screen
Init system (systemd/Winlogon) starts services · Login prompt presented

🗺️ Network Topologies VISUAL MODELS ▶
Diagram
Physical & Logical Network Topology Types
A network topology defines how devices are arranged and interconnected. Physical topology is the physical layout of cabling; logical topology is how data flows.
▸ TOPOLOGY VISUAL REFERENCE
Star
Central switch/hub. Easy fault isolation. Single point of failure.
Bus
Shared backbone. Simple but collisions & single failure = full outage.
Ring
Token-passing around a ring. One break = failure (unless dual-ring).
Full Mesh
Every node connects to every other. Max redundancy. n(n-1)/2 links.
Hybrid
Mix of topologies (e.g., star-bus). Used in most enterprise networks.
📡 DHCP Process — DORA 4-STEP SEQUENCE ▶
Sequence Diagram
Dynamic Host Configuration Protocol — DORA Handshake
DHCP automatically assigns IP addresses to clients. DORA describes the 4-message exchange between client and server. All messages use UDP (67/68). DHCP uses broadcast initially since the client has no IP.
▸ DORA SEQUENCE
D
Discover
Client broadcasts to find DHCP servers
→
O
Offer
Server offers available IP address
→
R
Request
Client requests the offered IP
→
A
Acknowledge
Server confirms IP lease to client
Key Details: Uses UDP 67 (server) & UDP 68 (client) · Default lease time typically 24h · APIPA fallback: 169.254.x.x · Relay agent (ip helper-address) routes DHCP across subnets

🔐 CIA Triad CORE MODEL ▶
Foundational Model
Confidentiality · Integrity · Availability
The three pillars of information security. Every security control, policy, and decision maps back to one or more of these properties.
▸ CIA TRIAD
🔒
Confidentiality
Data only accessible to authorized parties. Controls: encryption, access controls, MFA, classification
✅
Integrity
Data is accurate & unmodified. Controls: hashing, digital signatures, checksums, version control
⚡
Availability
Systems & data accessible when needed. Controls: redundancy, backups, DDoS mitigation, uptime SLAs
Extended: DAD Triad (attack perspective) — Disclosure (vs Confidentiality) · Alteration (vs Integrity) · Destruction/Denial (vs Availability)

⚔️ Cyber Kill Chain (Lockheed Martin) 7-PHASE MODEL ▶
Attack Framework
Adversary Intrusion Phases — Lockheed Martin Kill Chain
Models the 7 sequential stages of a targeted cyberattack. Defenders can break the chain by disrupting any phase. The earlier the disruption, the lower the adversary's ROI.
01
Reconnaissance
OSINT, scanning, target profiling. Passive (DNS, WHOIS) and active (port scanning)
02
Weaponization
Create exploit payload + delivery mechanism (malware, infected PDF, macro)
03
Delivery
Transmit weapon to target via email, USB, watering hole, web drive-by
04
Exploitation
Trigger exploit on target — CVE exploit, macro execution, social engineering success
05
Installation
Install RAT, backdoor, rootkit for persistent access. Modify registry, cron, services
06
C2 / Command & Control
Establish covert channel to attacker's C2 server (IRC, HTTP/S, DNS tunneling)
07
Actions on Objectives
Exfiltrate data, encrypt for ransom, lateral movement, destroy, espionage

🗂️ Linux Filesystem Hierarchy Standard (FHS) DIRECTORY TREE ▶
Diagram — Directory Tree
FHS — Standard Linux Directory Structure
The Filesystem Hierarchy Standard defines the directory structure of Linux/Unix systems. Everything begins at root /. Knowing the purpose of each directory is heavily tested on Linux+.
/ — Root of entire filesystem hierarchy
├─ bin/ — Essential user command binaries (ls, cp, mv, grep)
├─ sbin/ — System admin binaries (fdisk, iptables, init)
├─ etc/ — System-wide configuration files (passwd, fstab, hosts)
├─ home/ — User home directories (/home/alice, /home/bob)
├─ root/ — Root user's home directory (not inside /home)
├─ var/ — Variable data: logs (/var/log), spool, mail, databases
├─ tmp/ — Temporary files; cleared on reboot (usually)
├─ usr/ — User programs & data (secondary hierarchy)
├─ usr/bin/ — Non-essential user binaries
├─ usr/lib/ — Libraries for /usr/bin & /usr/sbin
└─ usr/share/ — Architecture-independent data (man pages)
├─ lib/ — Shared libraries for /bin & /sbin (libc.so, kernel modules)
├─ dev/ — Device files (block & character devices: /dev/sda, /dev/null)
├─ proc/ — Virtual FS: kernel & process info (/proc/cpuinfo, /proc/meminfo)
├─ sys/ — Virtual FS: hardware & kernel device info (sysfs)
├─ mnt/ — Temporary mount point for filesystems
├─ media/ — Auto-mounted removable media (USB drives, CDs)
├─ opt/ — Optional / third-party software packages
├─ srv/ — Service data (web server files, FTP data)
└─ boot/ — Boot loader files: kernel (vmlinuz), initrd, GRUB config

rwxrwxrwx — Permission Bit Structure
Linux permissions control read (r=4), write (w=2), and execute (x=1) for three user classes: Owner, Group, Others. Special bits: SUID (4), SGID (2), Sticky (1).
Type

4+2+1

EXAMPLE: -rwxr-xr--
File · Owner:rwx(7) · Group:r-x(5) · Other:r--(4) → chmod 754
SPECIAL BITS
SUID (4xxx): runs as file owner · SGID (2xxx): runs as group · Sticky (1xxx): /tmp — only owner can delete
