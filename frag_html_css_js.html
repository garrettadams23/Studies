🏷️ HTML — Structure & Semantic Elements WEB MARKUP ▶
Reference — HTML5
Document Structure · Semantic Tags · Forms · Meta · Accessibility
HTML defines structure, not style. Use semantic elements (<main>, <nav>, <article>) for accessibility and SEO. Always include lang, charset, and viewport.
▸ DOCUMENT BOILERPLATE
<!doctype html> <html lang="en"> <head> <meta charset="UTF-8"> <meta name="viewport" content="width=device-width, initial-scale=1.0"> <meta name="description" content="Page description for SEO"> <title>Page Title</title> <link rel="stylesheet" href="style.css"> </head> <body> <header><nav>...</nav></header> <main> <article>...</article> <aside>...</aside> </main> <footer>...</footer> <script src="script.js" defer></script> </body></html>
▸ SEMANTIC ELEMENTS
ElementPurpose
<header>Site/section header, logo, top nav
<nav>Primary navigation links
<main>Primary content — one per page
<article>Self-contained content (blog post, card)
<section>Thematic grouping with a heading
<aside>Supplementary content, sidebar
<footer>Bottom of page/section, links, copyright
<figure> / <figcaption>Image + caption pair
<details> / <summary>Native accordion / disclosure widget
<dialog>Native modal dialog
<time datetime="…">Machine-readable date/time
<mark>Highlighted/relevant text
<abbr title="…">Abbreviation with expansion

🎨 CSS — Selectors, Layout & Variables WEB STYLING ▶
Reference — CSS3 / Modern CSS
Selectors · Box Model · Flexbox · Grid · Custom Properties
Cascade order: !important > inline > ID > class/pseudo > element. Specificity: (0,1,0,0) = ID, (0,0,1,0) = class, (0,0,0,1) = element. Use custom properties for theming.
▸ SELECTORS & SPECIFICITY
p { } /* element (0,0,0,1) */ .card { } /* class (0,0,1,0) */ #hero { } /* id (0,1,0,0) */ a:hover { } /* pseudo-class (0,0,1,1) */ p::first-line { } /* pseudo-element (0,0,0,2) */ [type="text"] { } /* attribute (0,0,1,0) */ .nav > a { } /* child combinator */ .nav a { } /* descendant */ h2 + p { } /* adjacent sibling */ h2 ~ p { } /* general sibling */ :is(h1,h2,h3) { } /* :is() forgiving list */ :where(h1,h2) { } /* :where() — zero specificity */ :not(.hidden) { } /* negation */ :has(img) { } /* parent selector (modern) */
▸ CUSTOM PROPERTIES & CALC
:root { --color-primary: #00d4ff; --spacing-md: 1rem; --radius: 6px; } .card { color: var(--color-primary); padding: var(--spacing-md); width: calc(100% - 2rem); /* arithmetic */ font-size: clamp(1rem, 2.5vw, 2rem); /* fluid type */ }
▸ FLEXBOX
.container { display: flex; flex-direction: row; /* row | column */ flex-wrap: wrap; justify-content: space-between; /* main axis */ align-items: center; /* cross axis */ gap: 1rem; } .item { flex: 1 1 200px; /* grow shrink basis */ align-self: flex-start; order: 2; }
▸ CSS GRID
.grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(200px, 1fr)); grid-template-rows: auto; gap: 1.5rem; } .featured { grid-column: 1 / -1; /* span all columns */ grid-row: 1 / 3; }
⚡ JavaScript — ES2022+ Reference ES2022 · ES2023 · ES2025 ▶
Reference — Modern JavaScript
Variables · Functions · Async · DOM · Modern Features
Use const by default, let when reassignment is needed, never var. Modules use import/export. Always await inside async functions or use .catch().
▸ VARIABLES & DESTRUCTURING
const name = "Alice" let count = 0 // Array destructuring const [first, second, ...rest] = [1, 2, 3, 4] // Object destructuring const { name: n, age = 25 } = user // rename + default // Optional chaining (?.) — ES2020 const city = user?.address?.city // no throw if undefined const n = arr?.[0] // optional index // Nullish coalescing (??) — ES2020 const label = value ?? "default" // only falls back on null/undefined // Logical assignment — ES2021 a ||= "fallback" // assign if falsy a &&= transform(a) // assign if truthy a ??= "default" // assign if null/undefined
▸ FUNCTIONS & ARROWS
// Arrow — implicit return when no braces const double = x => x * 2 const add = (a, b) => a + b // Default params + rest function greet(name = "World", ...extras) { return `Hello, ${name}! Extras: ${extras.join(", ")}` } // Spread const merged = { ...defaults, ...overrides } const copy = [...arr, newItem] // Async / await — ES2017+ async function fetchUser(id) { try { const res = await fetch(`/api/users/${id}`) if (!res.ok) throw new Error(res.status) return await res.json() } catch (e) { console.error("Fetch failed:", e) } }
▸ ARRAY & OBJECT METHODS
// Array — non-mutating arr.map(x => x * 2) // transform each → new array arr.filter(x => x > 0) // keep matching → new array arr.reduce((acc, v) => acc + v, 0) // fold to single value arr.find(x => x.id === id) // first match or undefined arr.findIndex(x => x > 5) // first matching index arr.every(x => x > 0) // all pass? arr.some(x => x > 0) // any pass? arr.flat(2) // flatten 2 levels — ES2019 arr.flatMap(x => [x, x*2]) // map then flatten — ES2019 arr.at(-1) // last element — ES2022 // Object Object.keys(obj) // string[] of own keys Object.entries(obj) // [key,value][] pairs Object.fromEntries(entries) // reverse of entries — ES2019 Object.assign({}, src) // shallow clone/merge structuredClone(obj) // deep clone — ES2022
